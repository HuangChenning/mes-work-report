# 配色对比度规范

> 本文件定义了报告 HTML 生成中的配色规则。Agent 在 Step 4 生成 HTML 时读取此文件。

## 核心原则

**内联颜色必须使用 CSS 变量，禁止硬编码颜色值**（团队标识色除外）。

## 变量选择规则

| 元素类型 | 应使用的变量 | 禁止使用 |
|---------|------------|---------|
| 关键数据（满勤率、总工时等） | `--text-secondary-light` | `--text-tertiary-light`（太淡） |
| 辅助说明（"综合得分=XX"、"—"） | `--text-tertiary-light` | — |
| 警示信息（满勤率0%、未报工） | `--report-danger` + `font-weight:600` | `--text-tertiary-light` |
| 次要指标（2部均分等） | `--text-secondary-light` | `#888`（硬编码） |
| 主色调指标（业务已恢复等） | `--accent-primary` | `#00c7be`（硬编码） |
| 团队标识色（2部红/4部蓝/5部绿/整体橙） | `#ff3b30`/`#0071e3`/`#34c759`/`#ff9500` | —（允许硬编码） |

## 对比度检查清单

1. **深色卡片背景**（默认风格 `#16213e`）：`#555`、`#888` 等中灰色不可见，必须用 `var(--muted)` 或更亮的色
2. **白色卡片背景**（6个预设风格 `#ffffff`）：`--text-tertiary-light`（0.45透明度）用于关键数据会偏淡，关键数据用 `--text-secondary-light`（0.65透明度）
3. **深色背景区域**（hero等）：文字颜色必须用 `--text-primary-dark`/`--text-secondary-dark` 系列

## 默认风格 vs 预设风格的变量映射

| 变量 | 默认风格值 | 预设风格值 | 说明 |
|------|-----------|-----------|------|
| `--text-secondary-light` | `#888` | `rgba(0,0,0,0.65)` | 次要文字 |
| `--text-tertiary-light` | `rgba(255,255,255,0.45)` | `rgba(0,0,0,0.45)` | 辅助文字 |
| `--report-danger` | `#e94560` | `#ff3b30`/`#e74c3c`/`#e03e3e` | 危险色 |
| `--accent-primary` | `#00d4ff` | 各风格主色 | 主色调 |

## 设计规范

- **配色（默认风格）**：暗色主题，主背景 `#1a1a2e`，卡片 `#16213e`，强调色 `#e94560`（红）/ `#00d4ff`（青）
- **布局**：单列居中，最大宽度 1200px，卡片圆角 8px，阴影 `0 4px 6px rgba(0,0,0,0.3)`
- **表格**：宽度 100%，斑马纹（奇数行 `rgba(255,255,255,0.05)`），表头 `#0f3460`，hover 高亮
- **数值高亮**：满勤率 ≥ 80% 显示绿色，60%-80% 显示橙色，< 60% 显示红色
- **热力图**：使用 CSS 变量 `var(--hm-0-bg/text)` ~ `var(--hm-3-bg/text)`，各风格已预定义适配自身主题的配色（见下方热力图配色表）
- **响应式**：表格水平滚动，卡片自适应

## 热力图配色总览

热力图使用 4 个等级（0~3），每个等级有 `bg`（背景）和 `text`（文字）两个 CSS 变量。各风格已预定义适配自身主题的配色：

| 等级 | 含义 | 变量 |
|------|------|------|
| hm-0 | 0h / 无数据 | `--hm-0-bg` + `--hm-0-text` |
| hm-1 | 低于标准 60% | `--hm-1-bg` + `--hm-1-text` |
| hm-2 | 标准的 60%~100% | `--hm-2-bg` + `--hm-2-text` |
| hm-3 | 达标及以上 | `--hm-3-bg` + `--hm-3-text` |

**禁止在热力图中硬编码颜色**，必须使用 `var(--hm-X-bg)` 和 `var(--hm-X-text)`。

## 表格列宽规范

为防止单列过宽导致其他列换行，所有数据表格必须遵循：

1. **表格布局**：使用 `table-layout: auto`（浏览器根据内容自动分配列宽）
2. **表头**：`<th>` 默认 `white-space: nowrap`，但 `<td>` **不强制 nowrap**，允许长文本自然换行
3. **窄列加约束**（用 `max-width` 而非 `width`，配合 `overflow: hidden; text-overflow: ellipsis`）：
   - 姓名列 `.col-name`（max-width: 90px）
   - 城市列 `.col-city`（max-width: 80px）
   - 团队列 `.col-team`（max-width: 100px）
4. **数字列**：`.col-num`（width: 70px, `text-align: right`）、`.col-rank`（width: 40px, `text-align: center`）
5. **长文本列**（服务请求标题、报工内容、评价等）：不加宽度约束，允许自然换行
6. **禁止 `table-layout: fixed`**：会导致长文本溢出覆盖相邻列
